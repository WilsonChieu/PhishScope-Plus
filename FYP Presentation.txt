scan_history.json (current database file)

Currently the database will be kept on a local JSON file as the system runs entirely on localhost and has a single user which is me developing it. The JSON file acts as a lightweight stand-in for a proper database without needing to install any database server. 

The limitation here currently is it does not scale as repetition writes of data will make querying slow due to large datasets. How i would tackle this is , for real deployment I would use SQLite to manage the database since the current JSON file is already suited with SQLite format.
